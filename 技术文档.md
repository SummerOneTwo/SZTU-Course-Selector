# 技术文档 - SZTU 自动选课脚本

> 版本：v3.0 | 更新时间：2026-03

---

## 一、架构概览

```
sztu_course_selector.py
├── [模块加载与配置读取]     读取 config.toml 初始化课程分组 (Target Count 模式)
├── pad()                   DES 加密辅助函数
├── class Auth              登录 & 会话管理
│   ├── login()             执行完整的 SSO 登录流程
│   ├── logintoXK()         进入选课系统（登记批次 ID）
│   ├── get_course()        提交单次选课请求
│   ├── check_login()       校验当前登录态
│   ├── get() / post()      底层 HTTP 请求封装
│   └── encryptByDES()      DES 密码加密（静态方法）
├── select_course_worker()  单课程抢课线程函数
├── update_config_id()      将新批次 ID 写入 config.toml
├── wait_and_monitor()      等待指定时间后监控批次 ID
├── run_course_selection()  多线程抢课主循环（按组投递）
└── __main__                程序入口，串联完整流程
```

**核心设计原则：全程单 Session。** 登录只发生一次，监控与抢课两个阶段共享同一个 `Auth` 对象，避免被教务系统因重复登录而踢下线。

## 二、认证与登录流程

### 2.1 SSO 登录链路

教务系统使用深圳技术大学统一身份认证（SZTU IdP），登录流程为标准 SAML 重定向链：

```
① GET https://jwxt.sztu.edu.cn/
    → 302 → 教务系统登录导向
② GET 重定向链（共 3 跳）
    → 最终落地至 auth.sztu.edu.cn 认证页
③ GET https://auth.sztu.edu.cn/idp/AuthnEngine
④ GET https://auth.sztu.edu.cn/idp/authcenter/ActionAuthChain?entityId=jiaowu
⑤ POST ActionAuthChain（提交学号 + DES加密后的密码）
    → 校验 loginFailed == "false"
⑥ POST AuthnEngine（二次提交完成认证链）
    → 获得 SSO Location
⑦ GET SSO Location → 获得 jwxt.sztu.edu.cn 落地 URL
⑧ GET 落地 URL → 取得最终 JSESSIONID
⑨ GET xsMain.htmlx（确认登录成功，check_login 内校验 HTTP 200）
```

### 2.2 密码加密

使用 **DES-ECB** 模式 + PKCS5 Padding，密钥为 `PassB01Il71`（取前 8 字节）：

```python
def encryptByDES(message, key='PassB01Il71'):
    key1 = key.encode('utf-8')[:8]
    cipher = DES.new(key=key1, mode=DES.MODE_ECB)
    encrypted = cipher.encrypt(pad(message, block_size=8))
    return base64.b64encode(encrypted).decode('utf-8')
```

### 2.3 Cookie 结构

登录完成后，Session 中持有两个 `JSESSIONID`（新旧各一）以及 `SERVERID`，格式：

```
JSESSIONID={old};JSESSIONID={new};SERVERID={server}
```

## 三、监控模块 — `wait_and_monitor()`

### 3.1 流程

```
输入：auth_session（已登录）, known_ids（已知的无效批次 ID 列表）
  ↓
[监控阶段] 轮询 `https://jwxt.sztu.edu.cn/jsxsd/xsxk/xklc_list`
  ↓
用正则 r"(?:jx0502zbid=|toxk\(['\"])([A-Fa-f0-9]{32})" 扫描响应 HTML
  ↓
过滤 known_ids 中已知的旧 ID
  ↓
发现新 ID → 返回新批次 ID 字符串并写入配置
```

## 四、抢课模块 — `run_course_selection()`

### 4.1 课程分组与 Target Count

脚本通过 `config.toml` 加载**课程组（Course Group）**的概念：
- 每一门课（`kcid` 相同）作为一个分组。
- 该分组内可以配置多个不同的 `jx0404id`（即该门课的不同老师、不同时间段，只要抢到一个就算作该门课选课成功）。
- 停机条件取决于 `target_count` 参数，即当抢到的**组数**达到设定的 Target 时，自动退出脚本。

### 4.2 抢课循环逻辑

```
初始化：加载所有 active_groups = { 索引: 对应的课程参数组合 }

while active_groups 非空:
    遍历生成 tasks: 将所有组内的 jx0404id 全部取出进行投递
    使用 ThreadPoolExecutor(max_workers) 并发提交任务
    
    一旦组内某个 jx0404id 返回"成功"
    → 将该组从 active_groups 中删除，不再重试该组的其他 ID
    
    若本轮结束后总成功课程数 >= target_count
    → 任务达成，停止并发出系统级别的新进程弹窗提醒
    
    若仍有剩余 → 随机等待 [cool_down_min, cool_down_max] 秒，开始下一轮
```

### 4.3 选课请求

```
# 必修计划 (cno=0)
GET /jsxsd/xsxkkc/bxqjhxkOper?kcid={kcid}&cfbs=null&jx0404id={jxid}&xkzy=&trjf=

# 跨年级 (cno=1)
GET /jsxsd/xsxkkc/knjxkOper?kcid={kcid}&cfbs=null&jx0404id={jxid}&xkzy=&trjf=
```

响应为 JSON，通过 `message` 字段判断成功：包含 `"选课成功"` 字符串则视为成功。

## 五、配置系统 (config.toml)

放弃了早期 `config.txt` 与 `ConfigParser`，全面转用 Python 原生内置 `tomllib`，支持嵌套数据结构与数组：

```toml
[account]
username = "学号"
password = "密码"

[[courses]]
name = "高等数学"
kcid = "XXXXX"
cno = "1"
jx0404id = ["A_ID", "B_ID", "C_ID"]

[settings]
mode = "scavenge"
target_count = 1
jx0502zbid = "XXXXX"
max_workers = 8
round_cool_down_min = 280
round_cool_down_max = 320
schedule_start = "09:00"
schedule_end = "22:00"
```

| 字典 | 说明 |
|------|------|
| `account` | 认证密码配置 |
| `courses` | 每一个元素均为一门课程的参数列表 |
| `settings` | 全局限流与启停运行参数 |
