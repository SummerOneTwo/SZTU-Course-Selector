# SZTU 自动选课脚本

深圳技术大学教务系统自动化选课工具。**一键挂机**：脚本自动等待选课开放时间，在指定时刻抓取新批次 ID 并立即开始抢课，全程只需要一次登录。

## ✨ 功能特点

| 功能 | 说明 |
|------|------|
| 🔐 单次登录 | 全程复用同一 Session，不会被踢下线 |
| 📡 自动监控 | 在指定时间（默认 12:59）开始刷新，自动捕获新批次 ID |
| ⚡ 多线程抢课 | 使用线程池并发提交，可配置并发数 |
| 🔄 失败重试 | 失败课程自动进入下一轮重试 |
| 💾 自动更新配置 | 发现新批次 ID 后自动写入 `config.toml` |

## 📦 快速开始

### 1. 安装环境与依赖

> **对于 Windows 用户**：直接双击运行 `run.bat`，脚本会自动检测并提示您一键自动下载并安装 uv 环境，完全无需手动干预。

对于其他系统或偏好命令行的用户，请先手动安装 [uv](https://docs.astral.sh/uv/)：

```bash
uv sync
```

### 2. 初始化配置文件 (config.toml)

```bash
copy config.toml.example config.toml
```

编辑 `config.toml`，按实际情况填入：

```toml
[account]
username = "你的学号"
password = "你的教务系统密码"

[[courses]]
name = "大学物理实验B2"  # 日志和控制台显示的课名
kcid = "794AE26A58E546C593465C70E6FB9771"
cno = "1"                # "0"=必选计划, "1"=跨年级

# jx0404id 填写数组，代表同一门课的不同老师或时间段。一旦数组中任意一个投递成功，这门课即为“成功”并跳过投递
jx0404id = [
    "202520262001768",
    "202520262001256"
]

[settings]
mode = "scavenge"        # "monitor"=首抢模式, "scavenge"=捡漏模式
target_count = 1         # 期望抢到的【课程组】数量，达到了自动停止
jx0502zbid = "XXXX"      # 选课批次ID (捡漏模式必填)
max_workers = 8          # 并发数量
round_cool_down_min = 280
round_cool_down_max = 320
```

### 3. 获取课程参数

**获取 `kcid` 和 `jx0404id`：**
1. 登录教务系统，进入选课页面
2. 按 `F12` 打开开发者工具 → 切换到 **Network** 标签
3. 点击任意课程的「选课」按钮
4. 找到 `knjxkOper` 或 `bxqjhxkOper` 请求 → 在 URL 参数中读取 `kcid` 和 `jx0404id`
5. 如需查找其他老师班级，可使用本项目的 crawler.py 协助生成选课字典。

**关于 `jx0502zbid`（批次 ID）：**
- **推荐**：在“首发选课”模式 (`monitor`) 中可留空，脚本将在 `13:00` 开抢时自动获取最新批次
- **手动**：进入捡漏时，必须在开发者工具 Network 找到主文档或接口 URL 中的 `jx0502zbid=XXX` 并填入配置

## 🚀 使用方式

### 方式一：双击运行（推荐）

直接双击项目目录中的 **`run.bat`**，它会自动使用 uv 环境启动脚本。

### 方式二：命令行运行

```bash
uv run sztu_course_selector.py
```

### 运行流程

#### 首抢模式（`mode = monitor`）

```
启动 → 登录 → 等待 12:59 → 每 10s 刷新监控 → 发现新批次 ID → 自动抢课
```
适用于：选课系统尚未开放，需要第一时间抢课

#### 捡漏模式（`mode = scavenge`）

```
启动 → 登录 → 直接使用已有批次 ID → 反复轮询抢课（等别人退课）
```
适用于：选课系统已开放，课程已满，等待别人退课后捡漏

## 🔧 故障排除

| 问题 | 解决方案 |
|------|---------|
| 登录失败 | 检查学号/密码，确认账号未被锁定 |
| 无法找到配置文件 | 请复制 `config.toml.example` 生成 `config.toml` 文件 |
| 选课请求都返回“非主选已满” | 说明捡漏阶段目前无空位，脚本会自动计算冷却并轮询直到目标完成 |
| config 数组解析失败 | 请确保按照 toml 的标准，不要漏了双引号或方括号 |
